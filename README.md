# AI Code Reviewer

## AI Code Reviewer is a FastAPI backend that uses local LLMs (via Ollama and LangChain) to provide automated code reviews. It’s designed for developers who want quick, private feedback on their code, including explanations, issue detection, suggestions, and optimized code samples.

## Project Overview

""This project exposes a REST endpoint that accepts code (as a string) and a programming language, then returns a structured review generated by a local LLM. It’s ideal for:""

- Local code review automation
- Integrating into editors or CI pipelines
- Experimenting with LLM-based code analysis

## Setup & Installation

### Requirements:

- Python 3.14+
- Ollama running locally (default: http://localhost:11434)
- (Optional) PostgreSQL if you plan to use DB features

### Steps:

1. Clone the repo and navigate to the server directory:

```bash
cd f:/AI-Based-Projects/ai-code-reviewer/server
```

2. Create and activate a virtual environment:

```bash
python -m venv .venv
# PowerShell:
& .venv\Scripts\Activate.ps1
# or CMD:
.venv\Scripts\activate.bat
```

3. Install dependencies:

```bash
pip install -e .
# or, if you prefer:
pip install fastapi uvicorn langchain langchain-ollama ollama pydantic pydantic-settings sqlalchemy psycopg2-binary chromadb
```

## Configuration

Settings are managed in app/core/config.py and can be overridden with a .env file. Key options:

- APP_NAME: Service name (default: AI Code Reviewer)
- API_PREFIX: API base path (default: /api)
- API_VERSION: API version (default: v1)
- OLLAMA_BASE_URL: Ollama server URL (default: http://localhost:11434)
- LLM_MODEL: Model name (default: phi3:mini)
- DATABASE_URL: PostgreSQL connection string (not required for review endpoint)
- BACKEND_CORS_ORIGINS: CORS origins (list or string)

## Running the Server

From the server directory, start the FastAPI app:

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

You should see output indicating the server is running. Visit:

- http://127.0.0.1:8000/ for a status message
- http://127.0.0.1:8000/docs for interactive API docs (Swagger UI)

## API Usage

Endpoint

- POST /api/v1/code_review/

Request Body

```bash
{
  "code": "def add(a, b):\n    return a + b",
  "language": "python"
}
```

- code (string, required): The code to review
- language (string, optional): Programming language (default: "python")

## Response

```bash
{
  "review": "...LLM-generated review text..."
}
```

## How It Works

1. The API receives your code and language.

2. CodeReviewService builds a prompt for the LLM, asking for:

- Explanation
- Issues
- Improvements
- Optimized version

3. The prompt is sent to Ollama via LangChain.

4. The LLM response is returned as the review field.

## Development Notes

- The review endpoint is stateless and does not persist data by default.
- The service is designed for local/offline use; no code is sent to external APIs.
- You can extend the API by adding new endpoints in app/api/v1/ and new business logic in app/services/.
- The code review prompt is simple and can be customized in code_review_service.py.
- Example code snippets (like inefficient sorts, unused functions, etc.) are great for testing the review quality.
